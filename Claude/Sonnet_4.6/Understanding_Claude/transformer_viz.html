Input
Step 1 — Tokenization
Raw text is split into tokens and mapped to integer IDs from a vocabulary. Our tiny model has a vocabulary of 8 tokens.
Input sentence: "the cat sat"
Each word becomes a token ID by looking it up in the vocabulary table.
Vocabulary lookup
// vocabulary (simplified, real models have ~100k tokens)
vocab = { "the":0, "cat":1, "sat":2, "on":3, "mat":4, ... }

// "the cat sat" → token IDs
tokens = [0, 1, 2] // seq_len = 3

Real scale: Claude Sonnet processes sequences up to ~200,000 tokens. The vocabulary is ~100,000 subword tokens (using BPE — Byte Pair Encoding). "tokenization" itself becomes ["token", "ization"] — two tokens.
Embedding
Step 2 — Token Embedding
Each token ID is looked up in a learned embedding matrix E of shape [vocab_size × d_model]. Each token becomes a dense vector of floats.
Embedding lookup: E[token_id]

Real scale: In Claude, d_model ≈ 8,192. Each token becomes an 8,192-dimensional vector. The embedding matrix E has 100,000 × 8,192 = 819 million parameters — just for embeddings.
Positional
Step 3 — Positional Encoding
Transformers process all tokens in parallel — so they have no inherent sense of order. Positional encodings are added to inject position information.
Sinusoidal positional encoding (original Transformer)
PE(pos, 2i) = sin(pos / 100002i/d)
PE(pos, 2i+1) = cos(pos / 100002i/d)

// For pos=0 ("the"), d=4, i=0,1:
PE[0] = [sin(0), cos(0), sin(0), cos(0)] = [ 0.00, 1.00, 0.00, 1.00]
PE[1] = [sin(1), cos(1), sin(0.01), cos(0.01)] = [ 0.84, 0.54, 0.01, 1.00]
PE[2] = [ 0.91, -0.42, 0.02, 1.00]

// Final input to transformer:
X = E[tokens] + PE // element-wise addition

// "the" (pos 0):
X[0] = [ 0.50, -0.30, 0.80, 0.10] + [ 0.00, 1.00, 0.00, 1.00]
= [ 0.50, 0.70, 0.80, 1.10]
Modern models use RoPE (Rotary Position Embedding) instead — it encodes relative position by rotating Q and K vectors. Claude uses RoPE. The math is more complex but gives better results on long sequences.
Attention
Step 4 — Q, K, V Projections
Each token's vector is projected three ways — into a Query, a Key, and a Value — using three separate learned weight matrices.
Linear projections (one attention head, d_head=2)
W_Q, W_K, W_V ∈ ℝd_model × d_head = ℝ4×2

Q = X · W_Q // [3×4] · [4×2] = [3×2]
K = X · W_K // [3×4] · [4×2] = [3×2]
V = X · W_V // [3×4] · [4×2] = [3×2]

What Q, K, V mean intuitively
Q (Query) → // "What am I looking for?"
K (Key) → // "What do I contain / advertise?"
V (Value) → // "What information do I contribute?"

// Each token asks its Q against every other token's K
// High Q·K score = "this token is relevant to me"
// Then it collects those tokens' V proportionally
Real scale: d_head = 128 typically. With 64 heads, each layer has 3 × (8192 × 128) × 64 = ~201M parameters just in Q,K,V projection matrices per layer.
Attention
Step 5 — Attention Scores
Every token's Query is dot-producted with every token's Key to produce a score matrix. Scaled by √d_head to prevent vanishing gradients.
Scaled dot-product attention scores
scores = Q · KT / √d_head

// Q: [3×2], K^T: [2×3] → scores: [3×3]
// scaling: √2 ≈ 1.414

// scores[0][0] = Q[0]·K[0] / √2
= (0.56×0.42 + 0.70×0.31) / 1.414
= (0.235 + 0.217) / 1.414
= 0.452 / 1.414 = 0.320

Why scale by √d_head? Dot products grow in magnitude as dimension increases. Without scaling, softmax receives very large inputs, causing it to saturate (output nearly one-hot distributions), which gives near-zero gradients during training.
Softmax
Step 6 — Softmax & Attention Weights
Softmax converts raw scores into probabilities that sum to 1. These become the attention weights — how much each token "attends to" every other token.
Softmax formula
softmax(zi) = ezi / Σj ezj

// Row 0 of scores ("the" attending to all tokens):
z = [0.320, 0.518, 0.124]

// Exponentiate each:
e0.320 = 1.377, e0.518 = 1.679, e0.124 = 1.132

// Sum:
Σ = 1.377 + 1.679 + 1.132 = 4.188

// Normalize:
attn[0] = [0.329, 0.401, 0.270] // sums to 1.000 ✓

Temperature explained: At τ→0, the highest score gets all the weight (argmax). At τ→∞, all tokens get equal weight (uniform). Real sampling temperature works exactly this way — it's applied to the final output softmax to control how "creative" vs "deterministic" the model is.
Attention
Step 7 — Weighted Value Sum
The attention weights are used to take a weighted average of all Value vectors. This is the output of one attention head.
Attention output = softmax(QK^T/√d) · V
output[0] = 0.329 × V[0] + 0.401 × V[1] + 0.270 × V[2]

// V vectors (from V = X · W_V):
V[0] = [ 0.44, 0.28] // "the"
V[1] = [-0.12, 0.63] // "cat"
V[2] = [ 0.81, -0.19] // "sat"

= 0.329×[ 0.44, 0.28]
+ 0.401×[-0.12, 0.63]
+ 0.270×[ 0.81, -0.19]

= [ 0.145, 0.092]
+ [-0.048, 0.253]
+ [ 0.219, -0.051]

= [0.316, 0.294] // head output for "the"
Multi-head attention: This happens in parallel across all heads. Each head has different W_Q, W_K, W_V matrices so learns different patterns. The outputs are concatenated then projected: concat(head_1,...,head_h) · W_O. With 2 heads and d_head=2, the concat is [3×4], projected back to [3×4] = d_model.
Feed-Forward
Step 8 — Feed-Forward Network
After attention, each token's vector passes through a 2-layer MLP independently. This is where most of the model's "knowledge" is thought to live.
Position-wise FFN (applied to each token independently)
FFN(x) = GELU(x · W₁ + b₁) · W₂ + b₂

W₁ ∈ ℝd_model × d_ff = ℝ4 × 8 // d_ff = 4 × d_model typically
W₂ ∈ ℝd_ff × d_model = ℝ8 × 4

// GELU activation (smoother than ReLU):
GELU(x) ≈ x · Φ(x) // Φ = standard normal CDF
GELU(1.0) ≈ 0.841
GELU(-1.0) ≈ -0.159 // small but non-zero (unlike ReLU)
GELU(0.0) = 0.000

GELU vs ReLU comparison

Real scale: d_ff = 4 × d_model = 32,768 in large models. The FFN has 2 × (8192 × 32768) = ~537M parameters per layer. With 60 layers, FFN params alone ≈ 32 billion. This is where "facts" are believed to be stored as key-value memories.
Normalization
Step 9 — Layer Normalization
After each sub-layer (attention and FFN), the vector is normalized to have zero mean and unit variance, then scaled by learned parameters γ and β.
LayerNorm formula
LayerNorm(x) = γ × (x - μ) / √(σ² + ε) + β

// Example: x = [0.316, 0.294, -0.12, 0.88] after FFN
μ = (0.316 + 0.294 + (-0.12) + 0.88) / 4 = 0.343
σ² = mean of (x - μ)² = 0.118
σ = √0.118 = 0.344
ε = 1e-5 // numerical stability

// Normalize (before γ, β):
x̂ = (0.316 - 0.343) / 0.344 = -0.079 // first element

// γ, β are learned (initialized to 1, 0)
// Also: residual connection x = x + sublayer(x)
Residual connections are equally important — every sub-layer adds its output to its input: x = x + LayerNorm(Attention(x)). This lets gradients flow directly from output to early layers during training, solving the vanishing gradient problem and enabling very deep networks (60+ layers).
Output
Step 10 — Output Projection & Final Softmax
The final token vector is projected to vocabulary size, then softmax produces a probability distribution over all possible next tokens.
Logits and final softmax
// Unembedding: project d_model → vocab_size
logits = h · W_U // [1×4] · [4×8] = [1×8]

// Raw logits for our 8-token vocab:
logits = [2.1, 0.8, 1.5, 3.2, 0.3, -0.5, 1.1, 0.7]

// Apply softmax → probabilities:
probs = softmax(logits)

Token sampling: The model doesn't always pick the highest probability token (greedy). Techniques like top-p sampling (nucleus sampling) pick from the smallest set of tokens whose cumulative probability exceeds p (e.g. 0.9). This produces more varied, natural outputs than always choosing the argmax.
This completes one forward pass — producing one token. For a 100-token response, this entire pipeline runs 100 times. At real scale (billions of params, 200k context), each forward pass involves ~1015 floating point operations.
